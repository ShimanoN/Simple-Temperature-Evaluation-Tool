## 1. 背景・目的

現場でヒートガンや恒温槽・チャンバーを用いた温度評価業務の際、以下の課題が存在していました。

- 計測データが自動記録されない（手書きまたはExcel手入力が必要）
- 計測中の統計値（平均・ばらつき）が事後計算となっている
- リアルタイムで異常温度を検知する仕組みがない

これらの課題を解決するため、「計測から記録・統計計算までを自動化する専用ツール」の自主開発に取り組みました。

---

## 2. 実施内容

### 2-1. 企画・設計

- ハードウェア選定: M5Stack Core2/CoreS3（ESP32マイコン）+ K型熱電対 + MAX31855デジタル変換ICの組み合わせを採用
- パーツ調達・組み立て: 全部品を16,082円で調達。ハンダ付け不要設計により容易に組み立て可能な仕様
- ソフトウェアアーキテクチャ: 3タスク分離設計（IO、Logic、UIの責務を10ms/50ms/200msの周期で分担実行）
- 状態管理: IDLE（待機）→ RUN（計測中）→ RESULT（結果表示・保存）→ IDLEの循環構造

### 2-2. 機能実装

- リアルタイム温度計測: −200〜1350℃の広域計測に対応
- 統計値の自動計算: Welford法によるオンライン計算で、平均・標準偏差・最大・最小をリアルタイム更新
- CSV自動出力: MicroSDカードに10列形式で計測データを自動保存（Excel直接読み込み可）
- アラーム機能: 上限・下限温度超過を画面上で即時通知（EEPROMに設定値を永続保存）
- シンプル操作: ボタン1回押しで計測開始・終了（手順書不要）

**技術スタック**  
C++（Arduino フレームワーク）/ PlatformIO + VSCode / Git / GitHub

### 2-3. 品質確保

- テスト計画書の自作: 7カテゴリ（初期化・計測・終了・CSV形式・UI表示・エラー・パフォーマンス）33ケースを体系化
- 統合テストの全実施: 7カテゴリ33ケース全てを実行
- **テスト結果: 33/33 PASS（100%合格）**
