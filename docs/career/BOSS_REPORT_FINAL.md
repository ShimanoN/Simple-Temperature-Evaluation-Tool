## はじめに

## 1. 背景と課題

現場でヒートガン・恒温槽・チャンバーを使って温度評価を行う際、以下の非効率が存在していました。

- **データ記録が手動**: 計測値を逐一手書きするか、後からExcelに手入力する必要があった
- **統計値の計算が後作業**: 「平均温度」「ばらつき（標準偏差）」「最大・最小」は評価終了後に別途計算が必要だった
- **異常温度の検知が遅れる**: 上限・下限を超えた瞬間に気づく手段がなく、見逃しリスクがあった
- **記録フォーマットが不統一**: 担当者・回によって記録方法と集計フォーマットがばらつく

これらは「記録ミスのリスク」「評価1件あたりの工数」「データ品質の安定性」に直結する課題でした。

## 2. 開発した機能

### 機能一覧

| 機能 | 内容 |
|:---|:---|
| リアルタイム温度計測 | K型熱電対センサ使用、−200〜1350℃対応。計測値が画面にリアルタイム表示 |
| 統計値の自動計算・表示 | 計測中に平均・標準偏差・最大・最小を自動更新。計測終了後すぐに結果が確認できる |
| CSVファイルの自動保存 | MicroSDカードに計測データを自動記録。PCでExcel直接読み込み・グラフ化が可能 |
| アラーム機能 | あらかじめ設定した上限・下限温度を超えると画面で即時通知 |
| ワンボタン操作 | ボタン1回押すだけで計測開始・終了。手順書不要、誰でも使える |

### 使い方のイメージ（3ステップ）

```
① 電源ON → 現在の温度がリアルタイム表示される（待機状態）
② ボタンを1回押す → 計測開始。計測データがSDカードへ自動記録される
③ ボタンをもう1回押す → 計測終了。平均・ばらつき・最大・最小が画面表示、CSVが完成
```

SDカードをPCに挿すだけで、Excelで集計・グラフ化が完了します。

## 4. 将来の拡張性

現在の設計はメモリ使用量が少なく（プログラム領域31%・動作メモリ7%）、現状の約3倍の機能を追加できる余地が確保されています。業務ニーズに合わせて段階的に拡張可能です。

| 将来機能 | 内容 | 難易度見込み |
|:---|:---|:---:|
| タイムスタンプ | 計測日時の自動記録（現在は相対時刻のみ） | 約1週間 |
| クラウド送信 | WiFiによる計測データのリアルタイム転送 | 約1〜2週間 |
| グラフ表示 | 計測中の温度変化をリアルタイムにグラフ表示 | 約3〜5日 |
| 多チャンネル対応 | 複数箇所の温度を同時計測 | 約2〜3週間 |

## 5. 現状と今後の進め方

### 現在の位置づけ

本ツールは**設計・実装・品質検証が完了した試作版**です。実際のヒートガン・チャンバーを使った長期稼働試験はまだ実施していません。現場へ試行導入したうえで、運用上の課題があれば修正・改善を行う予定です。

### 他部門・他用途への応用

「センサの値をリアルタイムで記録・統計処理・アラーム表示する」という仕組みは、温度計測に限らず、湿度・圧力・電流など他の計測対象にも転用できます。同様のツールを必要とする他部署・他用途への横展開も、比較的低コストで実現可能です。
