## 2. 開発背景と課題認識

現場でヒートガンや恒温槽・チャンバーを用いた温度評価を実施する際、以下の運用上の非効率が存在していました。

- 汎用のデジタル温度計では、測定中の**時系列データを自動記録できない**
- 測定値を目視確認しながら手書き記録 or Excelへの手入力が必要
- 平均値・ばらつき（標準偏差）などの統計値は事後に別途計算が必要
- 測定中の異常温度（設定値超過）をリアルタイム検知する仕組みがない

これらの課題に対し、「計測から記録・統計計算まで一貫して自動化する専用ツール」があれば、1回の評価作業における工数を削減できると判断しました。

## 3. 開発成果

### 3-1. 完成したツールの機能概要

| 機能 | 内容 |
|:---|:---|
| リアルタイム温度計測 | K型熱電対を使用、−200〜1350℃の広域計測（画面表示 + 自動記録） |
| 統計値の自動計算 | 計測中に平均値・標準偏差・最大値・最小値をリアルタイムで更新 |
| CSVファイル自動出力 | 計測データをMicroSDカードへ自動保存（Excel直接読み込み可能） |
| アラーム機能 | 上限・下限温度の超過を画面上で即時通知（設定値はEEPROMで永続保存） |
| シンプル操作 | ボタン1回押しで計測開始 / 終了（手順書不要） |

### 3-2. ハードウェア構成とコスト

| 部品 | 役割 | 個数 |
|:---|:---|:---:|
| M5Stack Core2 | 制御・表示・入力 | 1 |
| K型熱電対センサ | 温度検出 | 1 |
| MAX31855モジュール | アナログ→デジタル変換 | 1 |
| MicroSDカード(16GB) | データ保存 | 1 |
| ジャンパワイヤセット | 配線 | 1 |
| **合計** | | **16,082円** |

ハンダ付け不要の完成品モジュールを組み合わせており、部品交換や修理が容易です。

### 3-3. 使用方法（3ステップ）

```
1. 電源ON → 温度がリアルタイム表示される（IDLE状態）
2. ボタン押下 → 計測開始（RUN状態）。データがMicroSDに自動記録される
3. ボタン押下 → 計測終了（RESULT状態）。平均・標準偏差・最大・最小を表示
```

計測データはMicroSDカードをPCに挿すだけでExcelで集計・グラフ化できます。

## 5. 開発プロセスで得た知見

本プロジェクトを通じて、以下のエンジニアリング経験を積みました。

- **C++組み込みプログラミング**: PLC（プログラマブルロジックコントローラ）での制御設計経験を基盤に、C++での組み込み制御設計を習得
- **通信プロトコル実装**: SPI・I2C通信を実装し、複数デバイスの同時制御を実現
- **デバッグ実践**: 5件の技術的課題を自力で根本原因まで追跡・修正
- **テスト設計**: 33ケースの検証計画書を自作し、系統的な品質確認を実施
- **ドキュメント整備**: 引き継ぎ可能な技術ドキュメント15ファイルを作成（GitHub公開）
