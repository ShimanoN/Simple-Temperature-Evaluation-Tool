## 【プロジェクト基本情報】（全タスク共通コンテキスト）

### プロジェクト概要

| 項目 | 内容 |
|:---|:---|
| **プロジェクト名** | Simple Temperature Evaluation Tool |
| **バージョン** | v1.0.0（開発終了・リリース完了） |
| **開発形態** | 個人開発（一人） |
| **開発期間** | 2026年2月～3月（約1ヶ月） |
| **リポジトリ** | https://github.com/ShimanoN/Simple-Temperature-Evaluation-Tool（公開、MIT） |
| **位置づけ** | 業務課題を解決するための自主開発ツール |

### 技術スタック

| カテゴリ | 内容 |
|:---|:---|
| **ハードウェア** | M5Stack Core2/CoreS3（ESP32） |
| **センサ** | K型熱電対 ＋ MAX31855 SPI/I2C デジタル変換 |
| **ストレージ** | MicroSD（CSV出力）、EEPROM（設定永続化） |
| **言語** | C++（Arduino フレームワーク） |
| **開発環境** | PlatformIO + VSCode |
| **バージョン管理** | Git / GitHub |
| **テスト** | 手動統合テスト 33ケース（自作テスト計画書） |

### 定量的実績（数値一覧）

| 数値 | 意味 |
|:---|:---|
| **33 / 33** | 統合テスト PASS率（100%） |
| **7カテゴリ** | I（初期化）D（計測）C（終了）F（CSV）U（UI）E（エラー）P（性能） |
| **Flash 31.0%** | ROM使用率（406KB / 1310KB）— 余裕率 69% |
| **RAM 7.2%** | RAM使用率（23KB / 327KB）— 余裕率 92.8% |
| **10ms / 50ms / 200ms** | 3タスクの実行周期（IO / Logic / UI） |
| **±1℃相当** | 計測精度（MAX31855 仕様） |
| **-200〜1350℃** | 計測可能範囲（K型熱電対使用時） |
| **16,082円** | ハードウェア総調達コスト |
| **3ファイル** | 主要ソースファイル数（main.cpp, Tasks.cpp, Global.h + 各 .h/.cpp） |
| **6モジュール** | 設計モジュール数（MeasurementCore / SDManager / EEPROMManager / DisplayManager / IOController / main） |

### 解決した技術的課題（バグ修正記録）

| 課題No | 課題 | 原因 | 解決策 |
|:---|:---|:---|:---|
| Bug-01 | RUN中 Average/StdDev が 0.0 のまま | 書き込み条件 `D_Count >= 2` で早期書き込み → Welford 未実行 | 書き込み条件を `D_Count >= 10`（100ms 待機）に変更 |
| Bug-02 | D_Average が計算されない | RESULT 遷移時のみ計算する設計バグ | Logic_Task（50ms周期）内でリアルタイム計算に変更 |
| Bug-03 | 温度異常値の混入 | MAX31855 readCelsius() の一時的なエラー値 | 最大3回リトライ処理を実装 |
| Bug-04 | SPI バス競合 | SD書き込み後の SPI バス状態不定 | 書き込み後 `SPI.end(); SPI.begin();` でリセット |
| Bug-05 | MicroSD 初期化失敗 | CS ピン設定ミス（GPIO 未指定） | microSD CS = GPIO4 に明示化 |

### プロジェクト開発者の背景

- **前職スキル**: PLC（プログラマブルロジックコントローラ）プログラミング経験あり
- **本プロジェクトでの成長点**: PLC的思考（スキャン、構造体、状態管理）をC++組み込み開発に適用
- **習得技術**: C++、ESP32アーキテクチャ、SPI/I2C通信、FreeRTOS準タスク設計、
  統計アルゴリズム実装、Gitによるバージョン管理、テスト設計・実行
- **ドキュメント**: 仕様書・実装ガイド・ハンドオーバーガイド・テスト計画書を全て自作（約15ファイル）

## 【出力ファイル仕様】

### Task 1: RAW_ACHIEVEMENTS.md

```
ファイルパス : docs/career/RAW_ACHIEVEMENTS.md
目的        : 全実績・技術事実の棚卸し素材（後続タスクの唯一の入力）
フォーマット : マークダウン（構造化リスト形式）
品質要件    : 事実のみ記載（解釈・言語化はしない）
              数値はすべて原文から引用（推測・丸めはしない）
              判断が必要な箇所は [要確認] とマークする
```

**必須セクション**:
1. `## 技術的事実（ハードファクト）` — 数値・仕様・ツール名
2. `## 実施した作業（行動事実）` — 設計・実装・デバッグ・テストの具体的行動
3. `## 解決した課題（問題解決）` — Bug-01〜05 の Before/After
4. `## 習得スキル（スキルマップ）` — 技術カテゴリごとに整理
5. `## ドキュメント成果` — 作成したドキュメント一覧と概要
6. `## 未解決事項・制限事項` — 正直な限界の記載
